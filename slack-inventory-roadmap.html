1. Purpose

Replace WhatsApp / Google Forms / manual chasing for inventory requests with a structured Slack application. EzyVet remains the system of record. Slack handles request → approval → routing → status, with a Google Sheet repository as the audit trail.

This is not an AI chatbot. It's a structured operational workflow — searchable product selectors, structured forms, decision buttons, notifications, status tracking.

3. Why these choices

Slack Bolt (vs raw API or alternatives)

  • Official framework, maintained by Slack
  • First-class support for modals (the request form), interactive buttons, block kit
  • Massive community + docs
  • Fits the Node-based Cheshire stack

Google Sheets (vs proper database)

  • Spec explicitly requires it for MVP
  • Ops team can edit user/location/category mapping without involving dev
  • Audit trail is human-readable
  • 31-column repository works fine up to ~10K rows; well beyond Year 1 volume
  • Easy migration to PostgreSQL when needed (same schema)

No EzyVet API in MVP

  • Risk-reduces MVP scope significantly
  • Procurement still creates POs/TOs in EzyVet the way they do today
  • PO/TO numbers entered back into Slack flow via "Mark as PO Created" button
  • Phase 2: does EzyVet have an API? If yes, automate. If no, keep manual loop.

Fuse.js for product search

  • Slack's native select menus don't fuzzy-search well across 2K+ items
  • In-memory Fuse index rebuilt every 5 min from product master sheet
  • Type "amox" → see "Amoxicillin 500mg Tablets"

5. Roadmap — phases & timeline

All dates assume kick-off on 2026-06-01.

Phase 0 — Pre-build setup Jun 01–07 (1 week)
  • Michael compiles product master export from EzyVet → Google Sheet
  • HR + Andrew compile user + Slack ID list per branch
  • AbdulRahman/Andrew creates the Slack app in Vet Alliance workspace + grants scopes
  • Andrew sets up GitHub repo + deploys hello-world Slack app to confirm infrastructure
  • Sinergi + Modern Vet kickoff call — align on Phase 1 scope

Output: all pre-reqs ready, hello-world app posts to a test channel.

Phase 1 — Core MVP Jun 08–28 (3 weeks)

Week 1 — Foundations

  • Slack app scaffolding (Bolt)
  • Google Sheets connector (read product master + user mapping into memory; write to request repository)
  • Fuse.js product search index, refreshed every 5 min
  • /inventory slash command opens a modal

Week 2 — Request form + repository

  • Modal: product search → quantity → urgency → branch → notes
  • On submit: write row to Google Sheet, generate Request ID, status=Submitted
  • Auto-fill SKU / Category / Subcategory / UOM / Assigned Approver from product master
  • Confirmation DM to requester

Week 3 — Approval flow

  • Approver receives interactive notification with Reject / Transfer / Purchase buttons
  • Each decision opens its own modal for required fields
  • "Prepared By" auto-filled from approver's Slack user

Output: single happy-path flow works end-to-end for one product / one approver / one branch.

Phase 2 — Operational queues + status tracking Jun 29 – Jul 12 (2 weeks)

Week 4 — Queue views

  • 3 operational queue views: Rejected / Transfer / Purchase
  • Filter by status, assigned approver, branch, urgency
  • "Mark TO Created" / "Mark PO Created" buttons → enter EzyVet reference
  • Status progression: Submitted → Approved → TO/PO Created → In Transfer/Ordered → Received → Closed

Week 5 — Notifications + edge cases

  • Slack notifications on every status change to requester
  • Reminders to approvers if pending >4h, >24h
  • Error handling: product not found, user not in mapping, sheet write failures
  • Admin slash command /inventory-admin for ops manager view

Output: full request lifecycle works. Ready for pilot.

Phase 3 — JLT pilot Jul 13–26 (2 weeks)
  • Week 6: Train JLT branch (15-min Loom + canvas in clinic-jlt-inventory). Train category managers (Pauline, Ivan, Aneesh). Daily check-ins, fix bugs in flight. Track requests/day, avg approval time, error rate.
  • Week 7: Tune based on real usage. Add quick fixes. Decide: ready for full rollout, or one more pilot week?

Output: pilot signed off by Michael.

Phase 4 — Full rollout Jul 27 – Aug 09 (2 weeks)
  • Roll out one cluster at a time: MV first, then BOV, then VV, then TVRC, then Mobile
  • Sunset WhatsApp inventory groups (Michael's announcement)
  • Full branch training (canvases + 15-min walk-through per clinic)
  • Monitor & support

Output: live across the group.

Phase 5 — Phase 2 enhancements Post Aug 10 (ongoing)

Candidates (not committed):

  • EzyVet API integration (auto-create PO/TO drafts)
  • Backup approver routing
  • Analytics dashboard: top requests, slowest approvers, branch volume
  • Slack workflow templates for repeat orders
  • Migration to proper database if SKU count >5K
