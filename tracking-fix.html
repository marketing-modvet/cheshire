1
Fix WP Rocket — Stop Delaying GTM
WP Admin → WP Rocket → File Optimization
🔴
This is the #1 priority. GTM currently loads delayed — phone clicks and WhatsApp clicks are invisible to GA4 and Ads. This one fix restores ~80% of your lost tracking.

Step-by-step

  1. Go to WP Rocket settings

    In WordPress admin, navigate to:

    WP Admin → Settings → WP Rocket → File Optimization

    Or use this direct URL:

    https://modernvet.com/wp-admin/admin.php?page=wprocket#file-optimization
  2. Find "Delay JavaScript Execution"

    Scroll down to the Delay JavaScript Execution section. It should be enabled (toggle is on). That's fine — we just need to exclude GTM and CallGear.

  3. Tick "Google Tag Manager" in the one-click list

    Under "3rd parties → Analytics & Trackers", tick the checkbox for Google Tag Manager. Google Analytics should already be ticked — leave it.

    Analytics & Trackers
    ✓
    Google Analytics
    ✓
    Google Tag Manager ← tick this
    Google Optimize
    HubSpot
  4. Add CallGear to manual exclusions

    Scroll further to find the "Excluded JavaScript Files" textarea. It currently contains JoiiChat.iife.js. Add app.callgear on a new line:

    Excluded JavaScript Files
    JoiiChat.iife.js app.callgear

    One entry per line. Don't remove JoiiChat.

  5. Save Changes

    Click the Save Changes button at the bottom of the page.

  6. Clear all caches

    Go to WP Rocket → Dashboard and click "Clear Cache". Also clear Cloudflare cache if enabled.

    WP Admin → WP Rocket → Dashboard → Clear Cache
  7. Verify in browser

    Open modernvet.com in a new incognito window → right-click → View Page Source → search for GTM-KRHZL582. It should appear inside a normal <script> tag, NOT as type="rocketlazyloadscript".

✅
Once done, GA4 and CallGear will both load immediately on page load. Phone click and WhatsApp tracking will be restored within hours.

GA4 — Cross-Domain Tracking
analytics.google.com → Admin → Data Streams
ℹ️
When a user goes from modernvet.com → book.floof.ae, GA4 currently treats it as two separate sessions. Cross-domain linking passes the session ID across so you can see the full journey in one session.

Set up cross-domain in GA4

  1. Open GA4 Admin → Data Streams
    analytics.google.com → Admin (bottom left) → Data Streams → modernvet.com (G-PDRSCBMJL0)
  2. Scroll to "Configure tag settings"

    Click "Configure tag settings" at the bottom of the stream settings page.

  3. Find "Configure your domains"

    Click "Configure your domains" — this is where cross-domain is set up.

  4. Add both domains
    Match type: Contains
    modernvet.com
    book.floof.ae

    Click Save.

  5. Repeat for book.floof.ae stream

    Go back to Data Streams → book.floof.ae (G-JE3Q11XD2D) → same steps, same two domains.

⚠️
The GTM Conversion Linker in bookme.modernvet.com (GTM-KMW364NM) currently lists bookme.modernvet.com as the linked domain. This should be book.floof.ae. Update it in GTM → bookme container → GAds Conversion Linker tag → linkerDomains → change to modernvet.com, book.floof.ae.

GA4 — Clean Up Duplicate Events
analytics.google.com + GTM-KRHZL582

Fix A — Remove duplicate payment tag in GTM

  1. Open GTM-KRHZL582 → Tags

    Find the tag "GA4 - Book successful payment" (id:15). It fires on ?tab=orders and sends book_successful_payment to the book.floof.ae stream (G-JE3Q11XD2D).

  2. Check if this event is needed

    The same event is tracked properly on bookme.modernvet.com container. This tag in the main container fires on a URL pattern (?tab=orders) that may not exist on modernvet.com anymore.

    Open the GTM Preview mode and check if it actually fires on any real page. If not — pause or delete it.

  3. Fix the stream if keeping it

    If you want to keep it, change measurementIdOverride from G-JE3Q11XD2D to G-PDRSCBMJL0 (main site stream).

Fix B — Disable outbound clicks (prevents double-firing)

  1. Go to GA4 → Admin → Data Streams → modernvet.com

    Click "Enhanced measurement" (the gear icon next to the toggle).

  2. Turn off "Outbound clicks"

    The Outbound clicks setting automatically fires on any link leaving the domain — including phone tel: links and WhatsApp links. Since you're tracking those manually with better event names, turn this off.

  3. Save
