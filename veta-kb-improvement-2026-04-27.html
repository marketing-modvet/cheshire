Classifier Key Change: Conditional Language ≠ Hedging

Removed from hedge patterns: depending, varies by, generally, typically, usually, sometimes. These describe reality (weight-based pricing, branch-specific rules) rather than genuine ignorance. True hedges remain: approximately, around AED, I think, not sure.

Phase 3 — Bot Logic Wiring query_kb.py v3.1

Three key updates to bot plumbing: DECISION header in query output, Audit-Class Playbook in SOUL.md, and escalation handoff stub.

3a. query_kb.py — DECISION Header

Every query now outputs a mandatory DECISION header before KB matches. The bot must follow it strictly. Two-layer detection: (1) user's own question checked for clinical/live/sensitive patterns, (2) KB match audit_class as fallback.

DECISION OutputBot BehaviourTrigger
DECISION: ANSWER from KB Use KB answer naturally. Speak as "we/our". Best match audit_class = CONFIDENT
DECISION: ESCALATE "Let me get the right person — give me a moment 💙" → call centre Clinical symptom/treatment in user question, or ESCALATE_CC KB match
DECISION: PLACEHOLDER (ezyVet) "Let me check our system — one moment! 🔍" Availability/schedule/stock in user question, or EZYVET KB match
DECISION: ACTION PLACEHOLDER Acknowledge booking/routing intent. Guide to action. BOT_ACTION KB match (booking, routing, link)
DECISION: REFUSE "We don't provide that directly — can connect you to customer service" Sensitive pattern in user question (nursing numbers, diagnostics pricing)
DECISION: ESCALATE (low confidence) "I'm not 100% sure — let me connect you with our team!" Best KB match distance > 0.55 and CONFIDENT class

Key fix: User-Question Override Layer

Previous version: DECISION based entirely on KB match audit_class → if a clinical question matched a non-clinical KB entry, it got ANSWER. Fixed: user's question is now checked independently for clinical/live-data/sensitive patterns before consulting KB match class. Clinical user question → always ESCALATE, regardless of what the KB matched.

3b. SOUL.md — Audit-Class Playbook

Appended "Audit-Class Playbook" section with behaviour table, trigger descriptions, and strict rule: "If DECISION says ESCALATE or REFUSE: DO NOT answer the substance of the question. Not even partial."
