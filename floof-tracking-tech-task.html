dataLayer Specification — Purchase Event

Fire this event immediately after TELR confirms a successful payment — before any page redirect. The same structure applies to all three brands; only the field values differ per booking. 

// Fire this on successful TELR payment (same structure for all 3 brands)
window.dataLayer = window.dataLayer || [];
window.dataLayer.push({
  'event': 'purchase',
  'ecommerce': {
    'transaction_id': 'TELR-12345',       // unique TELR transaction/order ID
    'value':          350,                   // actual service price in AED — varies per product
    'currency':       'AED',
    'affiliation':    'Modern Vet Hospital', // brand/clinic name (e.g. "Blue Oasis Vet", "Vienna Vet")
    'items': [{
      'item_name':  'Standard Consultation', // actual service name — varies per booking
      'price':      350,                    // same as value above
      'quantity':   1,
      'item_id':    'CONSULT-STD'           // optional: internal product/service ID
    }],
    'email':          'client@email.com',    // client email from booking form
    'phone_number':   '+971501234567'        // client phone from booking form
  }
});

Field reference

Field Type Required Notes
event string Required Must be exactly 'purchase' — this is the GTM trigger
ecommerce.transaction_id string Required Unique per booking — use TELR order/transaction ID
ecommerce.value number Required Actual price paid in AED — not a fixed amount, pulled from booking data
ecommerce.currency string Required Always 'AED'
ecommerce.items[0].item_name string Required Name of the service booked — pulled from booking data (e.g. "Standard Consultation", "Dental Cleaning")
ecommerce.affiliation string Recommended Brand/clinic name — helps distinguish multi-location data in GA4
ecommerce.email string Recommended Client email — used for enhanced conversions in Google Ads
ecommerce.phone_number string Optional Client phone in E.164 format — used for enhanced conversions
ℹ️
When to fire Fire the dataLayer push on the TELR payment success callback — ideally before any redirect to a confirmation page. If there is a dedicated confirmation/thank-you page, the push can also fire on page load of that page using data stored in the session or query params.

Modern Vet — Path Scoping

GTM-KRHZL582

Modern Vet tracking is already working. The only required change is to scope the GTM snippet to /modernvet pages only. Currently GTM-KRHZL582 loads globally on all book.floof.ae pages — this must be fixed before the other brand containers go live, otherwise Modern Vet's GA4 and Google Ads accounts will receive conversions from Blue Oasis and Vienna Vet bookings.

🚨
Do this first, before adding BOV and VV snippets Scoping MV to /modernvet is a prerequisite. Adding the other brand snippets without this fix will corrupt Modern Vet's conversion data.

Change required

Remove the global GTM-KRHZL582 snippet from the shared page template and load it conditionally:

// Load GTM-KRHZL582 ONLY on /modernvet pages
if (window.location.pathname.startsWith('/modernvet')) {
  // Google Tag Manager
  (function(w,d,s,l,i){w[l]=w[l]||[];w[l].push({'gtm.start':
  new Date().getTime(),event:'gtm.js'});var f=d.getElementsByTagName(s)[0],
  j=d.createElement(s),dl=l!='dataLayer'?'&l='+l:'';j.async=true;j.src=
  'https://www.googletagmanager.com/gtm.js?id='+i+dl;f.parentNode.insertBefore(j,f);
  })(window,document,'script','dataLayer','GTM-KRHZL582');
  // End Google Tag Manager
}

Or equivalently, if your routing is template-based, simply include the snippet only in the Modern Vet brand template/layout.

dataLayer push — no changes needed

The MV dataLayer push is already implemented. Verify it matches the specification in Section 2 above — particularly that value is the actual service price (not a hardcoded number) and items[0].item_name contains the service name.

Checklist

  • GTM-KRHZL582 removed from global template — loads only on /modernvet path
  • Verified GTM-KRHZL582 does NOT load on /blueoasis or /viennavet
  • dataLayer purchase event confirmed firing on TELR success
  • ecommerce.value is dynamic (actual price, not hardcoded)
  • ecommerce.items[0].item_name contains actual service name

Testing & Verification

✅
Test before going live Use GTM Preview + GA4 DebugView to verify each brand before deploying to production.

Method 1 — GTM Preview Mode

1
Open GTM → Container → Preview Enter https://book.floof.ae/[brandpath] and click Connect.
2
Verify page load tags fire On page load: GA4 Config tag and Conversion Linker should appear in the Tags Fired list.
3
Complete a test booking Trigger the TELR success flow. Check the GTM summary panel — you should see a purchase event appear, with both the GA4 Purchase tag firing.
4
Verify dataLayer values Click the purchase event → Data Layer tab. Confirm ecommerce.value is the correct price and ecommerce.items[0].item_name is the service name.

Method 2 — GA4 DebugView

Go to analytics.google.com → [Brand Property] → Admin → DebugView. Complete a test booking — the purchase event should appear within seconds with all parameters visible.

Cross-brand isolation check (critical)

Action Expected result
Complete booking on /blueoasis Appears in BOV GA4 only. NOT in MV or VV GA4.
Complete booking on /viennavet Appears in VV GA4 only. NOT in MV or BOV GA4.
Complete booking on /modernvet Appears in MV GA4 only. NOT in BOV or VV GA4.

Google Ads — Conversion Import from GA4

Once the first real bookings are tracked in GA4 (allow 24–48 hours after go-live), the marketing team will complete this step — no dev action required.

Steps (per brand, done in Google Ads UI)

1
Open the brand's Google Ads account Blue Oasis: 342-920-5760 · Vienna Vet: 503-678-3933
2
Goals → Conversions → New conversion action
3
Import → Google Analytics 4 → select purchase This imports the purchase event that GTM is now sending to each brand's GA4 property.
4
Set conversion value to "Use transaction-specific value" This uses the actual booking amount passed in the dataLayer.
