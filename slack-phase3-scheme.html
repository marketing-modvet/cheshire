Slack Lists — privacy-aware dual-list pattern (v1.4)

Important correction (May 7 v1.4)

Slack Lists permissions are tied to channel membership, not to views. A view filtered to "Branch = MB" lives inside the underlying list — anyone who can read the host channel sees the WHOLE list, the view just sorts/filters their default rendering.

Master + views does NOT give privacy isolation. Since most clinic channels are private and we want JLT staff to see only JLT's tasks, we use a dual-list pattern: per-clinic list inside each clinic channel + master aggregator list in the dept head's private team channel. Cheshire (or Slack workflow) syncs between them.

The dual-list pattern

clinic-mv-mb-maintenance       → list "MB maintenance"     (only MB staff + Judit)
clinic-mv-palm-maintenance     → list "Palm maintenance"   (only Palm staff + Judit)
clinic-mv-jlt-maintenance      → list "JLT maintenance"    (only JLT staff + Judit)
... (one per branch) ...

team-ops-maintenance           → list "Master Maintenance" (only Judit + maintenance team)
                                 ↑ Cheshire syncs all branch items here, tagged with branch

Result: JLT staff see only JLT's items. Period. Judit sees the master aggregator with everything. Cross-clinic patterns visible only to Judit — not to any individual branch.
